Robot Data Capture System

机器人数据采集平台

本项目用于多相机预览、夹爪数据采集、电动夹爪控制、点云采集、数据录制和 LeRobot/HDF5/RLDS 转换。当前主版本面向 Windows 10/11 x64，Linux 设备驱动层暂未完成统一适配。

后端C++14
前端原生 Web
默认端口8080
数据格式LeRobot/HDF5/RLDS

工程结构

目录/文件作用是否提交 Git
src/, include/C++ 后端、设备接入、录制、HTTP API必须提交
frontend/数据看板、采集控制台、说明页面必须提交
tools/数据转换脚本必须提交
lib/海康、Orbbec、GCAN、UMI SDK 或放置位置按授权决定，建议私有仓库提交
config.json默认端口、数据路径、采集参数提交默认模板
build/exe、DLL、构建缓存不提交
data_capture/原始采集数据不提交
data_converted/转换后的训练数据不提交
不要只复制 ManualGripper.exe。运行还需要 DLL、前端、配置、SDK 和转换脚本。

从零部署

1. 准备电脑

  1. 使用 Windows 10/11 64 位系统。
  2. 建议项目路径使用英文，例如 C:\Robot\ManualGripper。
  3. 准备管理员权限，用于安装相机、串口和 CAN 驱动。

2. 安装 MSYS2 和编译依赖

pacman -Syu pacman -S --needed mingw-w64-x86_64-gcc mingw-w64-x86_64-cmake mingw-w64-x86_64-make mingw-w64-x86_64-opencv mingw-w64-x86_64-eigen3 mingw-w64-x86_64-python

3. 安装 Python 转换依赖

python -m pip install --upgrade pip python -m pip install -r requirements.txt

4. 安装硬件驱动

  • 海康：安装 MVS 客户端和 USB/GigE 相机驱动。
  • Orbbec：安装 Orbbec SDK/Viewer/驱动。
  • UMI：插入后确认 Windows 设备管理器能看到 COM 口。
  • GCAN：安装 USBCAN 驱动，确认 CAN 线、电源和终端电阻。
  • IMX335：作为 UVC 摄像头使用，先用 Windows 相机 App 测试。

5. 检查 SDK 目录

lib/hikvision/include lib/hikvision/lib/win64/MvCameraControl.lib lib/orbbec/include lib/orbbec/lib/win64/OrbbecSDK.dll lib/gcan/ECanVci64.dll lib/gcan/CHUSBDLL64.dll

编译运行

MSYS2 MINGW64 编译

cd /c/Robot/ManualGripper mkdir -p build cd build cmake .. -G "MinGW Makefiles" mingw32-make -j4

PowerShell 编译

cd C:\Robot\ManualGripper $env:PATH = "C:\msys64\mingw64\bin;" + $env:PATH cmake -S . -B build -G "MinGW Makefiles" cmake --build build --config Release

启动服务

cd C:\Robot\ManualGripper\build .\ManualGripper.exe

启动后访问 http://localhost:8080。如果缺 DLL，回到项目根目录运行 .\collect_dlls.ps1。

操作流程

  1. 进入采集控制台：/index_old.html。
  2. 点击“重新扫描设备”，确认右上角设备信息和左侧开关都更新。
  3. 打开需要的彩色、深度、红外、点云或夹爪流。
  4. 进入“数据采集”，选择需要录制的数据项。
  5. 点击开始录制，完成动作后点击停止录制。
  6. 进入“数据转换”，选择会话并转换为 LeRobot/HDF5/RLDS。
  7. 回到数据看板检查历史记录。

海康只显示彩色流；Orbbec 显示彩色、深度、红外和点云。设备切换后如果左侧没有更新，先关闭相关预览流，再点击“重新扫描设备”。

数据格式

data_capture/20260602_103000/ ├─ Head-umi/ │ ├─ color_video/ │ ├─ depth_video/ │ ├─ ir-left_video/ │ ├─ ir-right_video/ │ └─ pointcloud_data/ ├─ Left-umi/ │ ├─ color_video/ │ └─ gripper_data/ ├─ Right-umi/ │ ├─ color_video/ │ └─ gripper_data/ └─ metadata.json

视频目录中保存 mp4 和 timestamps.csv。夹爪目录保存 gripper.csv。转换输出位于 data_converted/。

Git 提交

首次创建仓库

git init git status git add .gitattributes .gitignore README.md requirements.txt CMakeLists.txt config.json collect_dlls.ps1 setup_orbbec_sdk.ps1 docs frontend include src tools lib git commit -m "Initial data capture platform" git remote add origin <你的仓库地址> git branch -M main git push -u origin main

后续更新

git status git add README.md docs frontend include src tools config.json requirements.txt .gitignore .gitattributes git commit -m "Update deployment docs and device logic" git push

别人拉取

git clone <你的仓库地址> ManualGripper cd ManualGripper python -m pip install -r requirements.txt
提交前确认 build/、data_capture/、data_converted/ 没有进入待提交列表。

迁移打包

源码仓库版需要保留：

CMakeLists.txt config.json requirements.txt README.md docs/ frontend/ include/ src/ tools/ lib/

现场运行版可以额外保留 build/、data_capture/ 和 data_converted/，用于当前电脑直接运行和保留历史数据。

常见问题

页面打不开

确认 ManualGripper.exe 正在运行，默认端口为 8080。

右上角能看到设备，左侧没有开关

右上角显示检测列表，左侧显示已挂载的采集槽位。点击“重新扫描设备”，必要时先关闭旧预览流。

三路海康卡顿

降低 camera.maxWidth、camera.maxHeight 或 camera.fps。三路 USB 相机不建议满分辨率满帧率预览。

转换失败

确认已安装 numpy pyarrow h5py crcmod，并检查原始会话中是否有有效视频和 metadata.json。
